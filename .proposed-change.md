# Proposed Change: Direct Log Archival in Analyze Workflow

## Problem
- `state/agent_log.md` is 384KB / 571 lines — exceeds the 256KB single-read tooling limit, making weekly analysis increasingly unreliable
- `state/research_log.md` is 172KB / 1139 lines — approaching the same limit
- Both archival PRs (#39 agent log, #42 research log) were approved by the reviewer but are **merge-blocked by conflicts** — the same systemic pattern that has blocked cron fixes for 14 weeks
- Without archival, log files will continue growing and eventually break all workflows that read them

## Proposed Solution
Add a log archival step to `analyze.yml` (the weekly analysis workflow) that:
1. Moves `agent_log.md` entries older than 30 days to `state/agent_log_archive.md`
2. Moves `research_log.md` entries older than 30 days to `state/research_log_archive.md`
3. Commits both changes directly to main as AUTO-tier state file operations

## Why this approach
- Both log files and their archives are `state/` files — AUTO-tier per CLAUDE.md autonomy rules (no PR needed)
- The analyze workflow already runs weekly and reads both files — natural fit for maintenance
- Direct commit to main bypasses the merge conflict deadlock that has blocked PRs #39 and #42
- This is the same pattern that successfully resolved the FEATURE_STATUS.md deadlock (9 weeks of PR failures → direct commit)
- 30-day retention keeps recent context available while archiving historical entries

## Risk Assessment
- **Low risk**: Only moves content between state files, no behavioral changes
- **Reversible**: Archive files can be concatenated back if needed
- **Precedent**: FEATURE_STATUS.md direct commit (PR #34) established this pattern

## Files to Modify
- `.github/workflows/analyze.yml` — add archival step after analysis, before state commit
- Creates: `state/agent_log_archive.md`, `state/research_log_archive.md` (new files, append-only)

## Expected Impact
- Reduces agent_log.md from 384KB to ~50-80KB (keeping only last 30 days)
- Reduces research_log.md from 172KB to ~30-50KB
- Ensures both files remain within tooling read limits permanently
- Eliminates need for PRs #39 and #42 (can be closed)
