# Fix analyze.yml branch collision on same-day runs

## Problem
The "Create improvement PR" step in analyze.yml uses `analyze/$(date +%Y%m%d)` as the branch name. When analyze.yml runs multiple times in the same day (every 6 hours), the second run fails with a non-fast-forward push error because the branch already exists from the earlier run.

Currently at 1/3 consecutive failures (12:19Z on 2026-03-24). Next run at ~18:18Z will hit 2/3. Will reach 3/3 threshold and generate an issue within 24h.

## Root Cause
Line 123 of `.github/workflows/analyze.yml`:
```bash
BRANCH="analyze/$(date +%Y%m%d)"
```
Uses date only (no time component), so all same-day runs that produce a `.proposed-change.md` target the same branch.

## Proposed Solution
Change the branch name to include the GitHub Actions run ID for uniqueness:
```bash
BRANCH="analyze/$(date +%Y%m%d)-${GITHUB_RUN_ID}"
```

## Impact
- Low risk: only affects PR branch naming
- No behavioral change to analysis logic
- Prevents guaranteed future failures on same-day runs
