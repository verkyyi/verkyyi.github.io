## Problem
openai-harness-blog has been Cloudflare-blocked for 99+ consecutive days. Despite being
removed from evolve_config.md Research Sources (PR #14, 2026-03-26), the URL remains
hardcoded in evolve.yml workflow prompts. Every hourly evolve run checks this dead source,
generating ~4 wasted research_log entries per run (~168 per week, ~672 per month).

PR #65 (research-source-cleanup) attempted to fix this via the .proposed-change.md → PR
pathway but phantom-merged — only deleted .proposed-change.md without editing the workflow
YAML. This is the 8th phantom PR, confirming the .proposed-change.md approach cannot modify
workflow YAML files.

## Proposed Action
Create a GitHub issue requesting removal of the hardcoded openai-harness-blog reference
from the evolve.yml workflow prompt. The issue→triage→coder pathway is the only mechanism
proven to produce real changes (validated by #57→#58 in <3h on 2026-04-10).

The coder agent will need to open a PR (workflow YAML changes require needs-review per
CLAUDE.md autonomy rules). The PR may encounter merge conflicts from the hourly cron
state commits, but this is the best available pathway.

## Expected Impact
- Eliminate ~168 wasted research_log entries per week
- Reduce research_log growth rate by ~25%
- Remove one source of noise from evolve analysis runs
- Marginal reduction in API usage per run

## Risk
Low. Removing a dead research source has no functional impact. The source has returned
zero useful data in 99+ days.
