# Remove OpenAI blog from research sources in evolve_config

## Problem
The OpenAI harness blog is listed as a research source in `state/evolve_config.md`, but it has been 100% blocked by Cloudflare across 50+ fetch attempts over 4 days of operation. Every evolve run (~24/day) wastes API cycles attempting to fetch it, logging identical "blocked by Cloudflare challenge page" entries to the research log. This adds noise to logs and wastes context window tokens on every single evolve run.

## Root Cause
The OpenAI blog uses aggressive Cloudflare bot protection that blocks all non-browser HTTP requests. GitHub Actions runners will never pass this challenge. This is a permanent incompatibility, not a transient failure.

## Proposed Solution
Update `state/evolve_config.md`:
1. Remove `openai-harness-blog` from Research Sources
2. Rename `verkyyi/agentfolio` to `verkyyi/tokenman` (upstream repo was renamed)
3. Add `Version: v0.2.0` field to track upstream scaffold version

## Impact
- Eliminates ~24 failed fetch attempts per day
- Reduces research_log noise (currently ~6 identical "blocked" entries per research cycle)
- Corrects stale upstream reference (agentfolio→tokenman)
- No behavioral change to pipeline logic

- `state/evolve_config.md` (Research Sources section update)
