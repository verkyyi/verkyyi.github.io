# Proposed Change: Apply cron frequency reduction directly to workflow YAMLs

## Problem
The P0 cron frequency reduction has been recommended for 5 consecutive weeks but has NEVER been applied. Three PRs (#15, #16, #19) were created and merged/opened, but all only deleted the `.proposed-change.md` spec file without modifying the actual workflow YAML files.

Current cron schedules (UNCHANGED since bootstrap):
- evolve.yml line 5: `'0 * * * *'` (every hour at :00)
- watcher.yml line 5: `'30 * * * *'` (every hour at :30)

This produces ~48 state commits/day (93% of all git history), causing merge conflicts on every PR branch. PR #4 (landing page) has been blocked 191+ hours due to this.

**Root cause:** The proposed-change mechanism creates spec-only PRs. The evolve workflow opens a PR that deletes `.proposed-change.md` but never applies the file changes described in the "Files Changed" section.

## Proposed Fix
Directly modify the cron expressions:
- evolve.yml line 5: `'0 * * * *'` → `'0 */3 * * *'` (every 3 hours at :00)
- watcher.yml line 5: `'30 * * * *'` → `'30 */3 * * *'` (every 3 hours at :30)

Expected: ~16 state commits/day (down from ~48), reducing merge conflict frequency by ~67%.

## Files Changed
- `.github/workflows/evolve.yml` line 5: `'0 * * * *'` → `'0 */3 * * *'`
- `.github/workflows/watcher.yml` line 5: `'30 * * * *'` → `'30 */3 * * *'`

## IMPORTANT NOTE
Previous attempts to apply this change via the proposed-change mechanism failed because the mechanism only deletes this spec file without applying the changes. **The coder agent or human MUST directly edit the YAML files** for this fix to take effect.

## Risk
LOW — only changes scheduling frequency, no behavioral changes. Watcher health checks every 3h instead of 1h. Evolve still triggers on push via workflow_dispatch for urgent changes.
