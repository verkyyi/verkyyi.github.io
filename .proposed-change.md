# Proposed Change: Add conditional state commits to reduce commit noise

## Problem
evolve.yml and watcher.yml commit state files on every run (~24 runs/day each),
producing ~48 state commits/day even when nothing changed. This creates merge
conflicts on every PR branch within hours, making it impossible for the
autonomous pipeline to merge workflow YAML changes (6 failed attempts over 6
weeks). The cron frequency fix is blocked on human action, but this change
reduces the blast radius at the current frequency.

## Change
In both `evolve.yml` and `watcher.yml`, wrap the state commit step with a
conditional check:

```yaml
- name: Commit state changes
  run: |
    git add state/
    if git diff --cached --quiet; then
      echo "No state changes to commit"
    else
      git commit -m "state: $COMMIT_MSG"
      git push
    fi
```

This skips commits when state files haven't actually changed, which happens
on ~30-40% of runs (watcher "all clear" with identical content, evolve runs
with no new findings).

## Expected Impact
- Reduces state commits from ~48/day to ~30/day (estimated 35% reduction)
- Extends PR branch viability window from ~30min to ~45min
- Does not fix the root cron issue (human-only) but reduces downstream damage
- Zero risk: only skips no-op commits, all actual state changes still committed

## Type
workflow YAML change — requires needs-review PR

## Priority
P1 — mitigates the P0 cron deadlock while waiting for human intervention
