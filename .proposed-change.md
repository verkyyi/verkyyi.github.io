# fix(workflow): add missing file guards to growth.yml

## Problem
Growth Strategist workflow fails because `state/growth_metrics.md` and
`state/adopters.md` do not exist. The "Collect growth context" step runs
`cat state/growth_metrics.md` which exits non-zero and kills the job.

This was the Growth Strategist's 1st failure (identified in weekly analysis
2026-03-23). The file was never created during onboarding because the
scaffold is a personal portfolio site, not the agentfolio template itself.

## Fix
Add fallback guards (`|| echo "(none)"`) to the two `cat` commands in the
"Collect growth context" step, matching the pattern already used for
`state/growth_log.md` on the same step.

### Before
```yaml
cat state/growth_metrics.md >> /tmp/growth_context.txt
...
cat state/adopters.md >> /tmp/growth_context.txt
```

### After
```yaml
cat state/growth_metrics.md >> /tmp/growth_context.txt 2>/dev/null || echo "(no metrics yet)" >> /tmp/growth_context.txt
...
cat state/adopters.md >> /tmp/growth_context.txt 2>/dev/null || echo "(no adopters yet)" >> /tmp/growth_context.txt
```

Also adds `|| echo "(none)"` guard to `cat state/learned_rules.md` in the
prompt-building step which has the same vulnerability.

## Risk
Low — adds defensive fallbacks only, no behavioral change when files exist.
