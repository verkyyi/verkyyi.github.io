# Proposed Change: Reduce evolve.yml and watcher.yml cron to every 3 hours

## Problem
State commits account for 97% of all git history (534/551 commits this week). The root cause is hourly cron schedules in evolve.yml (`0 * * * *`) and watcher.yml (`30 * * * *`), producing ~96 state commits/day. This causes merge conflicts on every PR branch, blocking all forward progress. PR #4 (landing page) has been stuck for 209+ hours.

PR #15 (merged 03-26) was titled "Reduce evolve.yml and watcher.yml cron frequency" but did NOT change the actual YAML cron expressions. PRs #16, #19, #22, #23 all attempted to fix this but none were merged (reviewer failures, merge conflicts).

This is the 5th consecutive week this has been identified as the #1 systemic bottleneck.

## Changes
- `evolve.yml` cron: `0 * * * *` → `0 */3 * * *` (every 3 hours)
- `watcher.yml` cron: `30 * * * *` → `30 */3 * * *` (every 3 hours, offset from evolve)

## Expected Impact
- State commits reduced from ~96/day to ~32/day (67% reduction)
- Merge conflict probability drops proportionally
- CI minutes reduced ~67%
- Aligns with tokenman v0.3.0 "Lean Operations" approach (69% cost reduction via same optimization)

## Risk
- Health check frequency drops from hourly to every 3h — issues detected up to 2h later
- Evolve research checks less frequent — acceptable since sources show minimal activity
