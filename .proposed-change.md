# Proposed Change: Research Log Rotation

## Summary
Archive research log entries older than 14 days to `state/research_log_archive.md`, keeping the active `state/research_log.md` under ~500 lines.

## Rationale
The research log has grown to 1073 lines (62K tokens), which **exceeds the Read tool's 25K token limit**. This actively degrades the weekly analysis workflow — it can no longer read the full research log to analyze findings from the past 7 days. The agent log (523 lines) is approaching the same limit.

PR #40 (research log quiet-run aggregation) reduced the *growth rate* by ~75%, but the existing backlog remains too large. This is the same concept as the approved-but-blocked PR #39 (agent log archival), applied to the larger and more urgent file.

## Changes
1. **`.github/workflows/analyze.yml`** — Add a rotation step before analysis:
   - Count lines in `state/research_log.md`
   - If >500 lines, move entries older than 14 days to `state/research_log_archive.md` (append)
   - Keep header and recent entries in active file
2. **`state/research_log_archive.md`** — New file, append-only archive of rotated entries

## Implementation
```bash
# Pseudocode for rotation step
HEADER=$(head -3 state/research_log.md)
CUTOFF_DATE=$(date -d '14 days ago' +%Y-%m-%d)
# Keep lines with dates >= CUTOFF_DATE + header
# Append older lines to research_log_archive.md
```

## Risk Assessment
- **Low risk**: Append-only archive, no data loss
- **Reversible**: Archive can be concatenated back if needed
- **Scope**: Only affects analyze.yml workflow and state/ files (AUTO-tier)
- **Precedent**: Same approach as PR #39 (agent log archival), which was approved by reviewer
