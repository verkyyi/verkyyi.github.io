# Proposed Change: Agent Log Compaction for Routine Health Checks

## Problem
The watcher workflow runs hourly and writes a 400-600 character entry to agent_log.md for every health check, even when all systems are healthy and zero corrective actions are taken. This produces ~24 entries/day, growing agent_log.md to 400+ lines (272KB+) within 9 weeks. The file is now too large to read in a single tool call, degrading analysis quality.

## Proposed Solution
When the watcher health check finds:
- 0 corrective actions
- All workflows healthy
- No broken chains or stuck runs

Write a single condensed line instead of the current verbose format:

**Current format (~500 chars):**
```
2026-03-30T11:52Z | watcher.yml | health check | all clear — 0 corrective actions. All workflows healthy: Self-Evolve (11:24Z), pages-build-deployment (11:26Z), ... [full status of every workflow, every PR, every issue, token utilization stats]
```

**Proposed format (~80 chars):**
```
2026-03-30T11:52Z | watcher.yml | health check | all clear — 0 actions
```

Only expand to full verbose format when:
- Corrective actions are taken (>0)
- A workflow is unhealthy or degraded
- A broken chain or stuck run is detected
- Token utilization shows anomalies

## Impact
- Reduces agent_log growth by ~80% for routine operations
- Keeps full detail for all actionable events
- Makes agent_log readable in a single tool call again
- Reduces state commit sizes (smaller diffs)

## Risk
- LOW: No behavioral change to watcher — same checks, same corrective actions
- Only the logging verbosity changes for "all clear" results
- Full detail preserved for any non-routine event

## Tier
PR — auto-merge (minor skill improvement, behavioral, low-risk)
