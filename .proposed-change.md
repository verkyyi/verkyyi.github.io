# Proposed Change: Agent Log Archival

## Summary
When `state/agent_log.md` exceeds 200 entries, archive entries older than 14 days
to `state/agent_log_archive/` (one file per ISO week, e.g. `week-2026-W12.md`).
Keep the working log lean while preserving full history.

## Motivation
The agent log is currently 483 lines and growing ~70 entries/day (primarily watcher
health checks). Even with compaction (PR #35) and abbreviated format (PR #38), the
log will exceed 1000 lines within 2 weeks. Large state files increase commit diff
noise and slow down session start (every workflow reads agent_log.md on start).

The research log already hit 1018 lines — agent_log is on the same trajectory.

## Implementation
1. At the start of each `analyze.yml` run (weekly):
   - Count entries in `agent_log.md`
   - If > 200 entries, move entries older than 14 days to `state/agent_log_archive/week-YYYY-WXX.md`
   - Keep the `# Agent Log` header and `<!-- Append entries below -->` marker
   - Archive files are append-only, never modified after creation
2. Archive directory is AUTO-tier (state file management)
3. Weekly summaries are always kept in the working log (never archived)

## Impact
- Reduces agent_log.md from ~500 lines to ~200 lines (60% reduction)
- Preserves full audit trail in archive files
- Runs once per week (minimal overhead)
- No behavioral changes to any workflow

## Risk
LOW — purely organizational change to state files. Archive files are read-only
after creation. Working log behavior unchanged. Rollback: move archive entries
back to agent_log.md.

## Tier
AUTO (state file management) — or PR with auto-merge if preferred.
