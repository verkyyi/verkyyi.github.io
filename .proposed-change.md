# Proposed Change: Emergency Log Truncation via Direct State Commit

## Problem
Both log files have exceeded tooling limits and are now operationally degraded:
- **research_log.md**: 260KB — EXCEEDED 256KB Read tool limit. The file is now **unreadable** by the automation that depends on it. Every evolve run that appends to it makes the problem worse.
- **agent_log.md**: 563KB — 2.2x the 256KB tooling limit. Can only be read in chunks, degrading analysis quality.

5 PRs attempting to fix this (#39, #42, #48, #50, #51) are ALL merge-blocked by conflicts caused by hourly state commits. This has been the case for 5+ weeks with zero progress.

## Proposed Solution
The analyze workflow should perform emergency inline truncation of both files as an AUTO-tier state file operation:

1. **agent_log.md**: Keep header + last 100 entries (from current 867 lines). Prepend a truncation notice with the date and count of removed entries.
2. **research_log.md**: Keep header + last 200 entries (from current 1722 lines). Prepend a truncation notice.

This reduces agent_log to ~60KB and research_log to ~30KB — well within tooling limits and providing 4-8 days of historical context.

## Justification
- State file updates are AUTO tier per CLAUDE.md autonomy rules
- research_log is NOW unreadable — this is not a future risk, it's a current failure
- 5 PR-based solutions have all failed (merge-blocked 5+ weeks)
- The analyze workflow already has inline truncation (PR #49 merged 04-04) but thresholds are too conservative
- Log entries are append-only operational records, not user content — truncation loses history but not functionality
- The alternative (waiting for human) has produced zero action in 35+ days

## Implementation
In analyze.yml, after the analysis step:
```bash
# Emergency truncation when logs exceed tooling limits
if [ $(wc -c < state/agent_log.md) -gt 262144 ]; then
  HEADER=$(head -3 state/agent_log.md)
  TAIL=$(tail -100 state/agent_log.md)
  echo "$HEADER" > state/agent_log.md
  echo "<!-- TRUNCATED $(date -u +%Y-%m-%dT%H:%MZ): kept last 100 entries -->" >> state/agent_log.md
  echo "$TAIL" >> state/agent_log.md
fi
if [ $(wc -c < state/research_log.md) -gt 262144 ]; then
  HEADER=$(head -3 state/research_log.md)
  TAIL=$(tail -200 state/research_log.md)
  echo "$HEADER" > state/research_log.md
  echo "<!-- TRUNCATED $(date -u +%Y-%m-%dT%H:%MZ): kept last 200 entries -->" >> state/research_log.md
  echo "$TAIL" >> state/research_log.md
fi
```

## Risk
- Loss of historical log entries (mitigated: all entries exist in git history)
- Truncation may remove context needed for future analysis (mitigated: 100/200 entries covers 4-8 days)
