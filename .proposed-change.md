# Proposed Change: Evolve No-Action Run Compaction in Agent Log

## Summary
When an evolve.yml run takes no action (research all-quiet, pipeline 0-actionable, 0 issues created), write a single abbreviated line to agent_log.md instead of the current 200-400 character verbose entry.

## Motivation
- Evolve runs ~24 times/day (hourly cron), producing agent_log entries that are ~50% of all log lines
- ~95% of evolve runs take no action and report identical patterns: routine research, 0 pipeline issues, 0 human intent, HUMAN_ACTIVE skip
- Agent log grew from 400→509 lines this week despite compaction PR #35 (watcher) — evolve entries are the remaining growth driver
- This follows the successful pattern of PR #35 (watcher health check compaction, ~80% reduction) and PR #40 (research log quiet-run aggregation)

## Proposed Format
**Current** (no-action evolve, ~300 chars):
```
2026-04-01T14:35:05Z | evolve.yml | self-evolution run (HUMAN_ACTIVE) | Research: all-quiet (aggregated). tokenman routine state, quarto-cli lua-types docs PR#14295, openai blocked, pipeline 0-actionable. Human intent: 0 in 7d. No agentfolio release. Config recheck within 7d. Hour 14 UTC (skipped 2c/2d/2e). PR #40 merged (quiet-run aggregation). 0 issues created per HUMAN_ACTIVE. System stable.
```

**Proposed** (no-action evolve, ~120 chars):
```
2026-04-01T14:35Z | evolve.yml | no-action | research:quiet pipeline:0 human:0 HUMAN_ACTIVE. System stable.
```

**Action runs** (when evolve actually does something) remain verbose with full detail.

## Expected Impact
- ~40-50% reduction in agent_log line growth (evolve is ~50% of entries, ~95% are no-action)
- Consistent with existing compaction patterns (PRs #35, #40)
- Preserves full detail for actionable runs

## Risk
Low — same pattern as two previously merged compaction PRs. Read-back unaffected (analysis and watcher already parse structured fields).

## Autonomy Tier
PR — label: auto-merge (minor skill improvement, behavioral, low-risk — consistent with PR #35 and #40 precedent)
