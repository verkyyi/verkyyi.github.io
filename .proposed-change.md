# Proposed Change: Reduce evolve.yml and watcher.yml cron to every 3 hours

## Problem
Hourly cron schedules in evolve.yml and watcher.yml produce ~96 state commits/day (604/week, 97% of all commits). This is the #1 systemic bottleneck — it causes merge conflicts on every PR branch, blocking all forward progress. 7 PRs have attempted this fix (#15, #16, #19, #22, #23, #25, #26) over 6 weeks. NONE modified the actual YAML cron expressions. The coder agent consistently edits state files but fails to change workflow YAML.

**Current values (MUST be changed):**
- `.github/workflows/evolve.yml` line: `    - cron: '0 * * * *'`
- `.github/workflows/watcher.yml` line: `    - cron: '30 * * * *'`

## Changes
**File 1: `.github/workflows/evolve.yml`**
- Find: `- cron: '0 * * * *'`
- Replace with: `- cron: '0 */3 * * *'`

**File 2: `.github/workflows/watcher.yml`**
- Find: `- cron: '30 * * * *'`
- Replace with: `- cron: '30 */3 * * *'`

**CRITICAL: This change MUST modify the two `.yml` files above. A PR that only updates state/ files is NOT a fix. Verify the PR diff includes changes to `.github/workflows/` before merging.**

## Expected Impact
- State commits reduced from ~96/day to ~32/day (67% reduction)
- Merge conflict probability drops proportionally — may unblock PR #4 (230+h stale)
- CI/CD minutes reduced ~67%
- Aligns with tokenman v0.3.0 "Lean Operations" approach (69% cost reduction via same optimization)

## Risk
- Health check frequency drops from hourly to every 3h — issues detected up to 2h later
- Evolve research checks less frequent — acceptable since sources show minimal activity
- LOW overall risk: watcher still runs every 3h, issues still auto-detected within one cycle
