# Proposed Change: Fix watcher-created issues missing auto-triage

## Problem
When the watcher workflow creates issues (via GitHub API), the `triage.yml` workflow
is not triggered because `issues: [opened]` events from GITHUB_TOKEN do not trigger
other workflows (GitHub Actions anti-recursion rule). This causes a 2-3 hour delay
while the watcher detects the untriaged issue and manually re-triggers triage via
`workflow_dispatch`.

Confirmed pattern: both issue #6 (Growth Strategist 3/3) and issue #8 (Weekly
Analysis 3/3) required manual triage re-trigger by watcher after 2-3 hours.

## Root Cause
GitHub Actions prevents workflows triggered by GITHUB_TOKEN from triggering other
workflows to avoid infinite loops. When watcher creates an issue, the `issues: opened`
event fires but triage.yml ignores it because the actor is `github-actions[bot]`.

## Proposed Solution
Add a `workflow_dispatch` call from watcher to triage immediately after creating an
issue, so the full pipeline (triage -> coder -> reviewer) starts without delay.

### Implementation sketch (watcher.yml addition)
```yaml
# After creating an issue in watcher.yml
- name: Trigger triage for new issue
  if: steps.create_issue.outputs.issue_number
  run: |
    gh workflow run triage.yml \
      -f issue_number=${{ steps.create_issue.outputs.issue_number }}
  env:
    GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Impact
- Eliminates 2-3 hour triage delay on watcher-created issues
- Reduces watcher complexity (no need for "check if untriaged" logic)
- Full pipeline (triage->coder->reviewer->merge) would complete in ~5 min instead of ~3h

## Alternatives Considered
1. **Use PAT instead of GITHUB_TOKEN** — would trigger events but introduces secret
   management overhead and security risk
2. **Inline triage logic in watcher** — duplicates triage workflow, harder to maintain
3. **Keep manual re-trigger** — current approach, adds 2-3h latency (unacceptable)

## Risk
Low — workflow_dispatch is a standard mechanism. Triage already handles being called
for already-triaged issues gracefully.
