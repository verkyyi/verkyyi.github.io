# Proposed Change: Reduce evolve.yml and watcher.yml cron frequency

## Problem
State commits account for 96% of all git history (~48 commits/day from hourly evolve + hourly watcher). This is the root cause of merge conflicts on every PR branch, blocking all forward progress. PR #15 was intended to fix this but only deleted the previous .proposed-change.md — the actual YAML files were never modified. This is the 5th consecutive week this has been the top priority.

## Solution
Reduce cron frequency from every hour to every 3 hours for both workflows:
- evolve.yml: `0 * * * *` → `0 */3 * * *` (8 runs/day instead of 24)
- watcher.yml: `30 * * * *` → `30 */3 * * *` (8 runs/day instead of 24)

This reduces state commits from ~48/day to ~16/day (67% reduction).

## Changes Required

### .github/workflows/evolve.yml
```yaml
# Before:
    - cron: '0 * * * *'
# After:
    - cron: '0 */3 * * *'
```

### .github/workflows/watcher.yml
```yaml
# Before:
    - cron: '30 * * * *'  # Every hour at :30 (offset from evolve at :00)
# After:
    - cron: '30 */3 * * *'  # Every 3 hours at :30 (offset from evolve at :00)
```

## Risk Assessment
- **Low risk**: Both workflows also support workflow_dispatch for on-demand runs
- **Monitoring gap**: Health checks go from hourly to every 3h — acceptable since all workflows are currently healthy and the watcher has had 0 actionable findings in the last 48h
- **Evolve gap**: Self-evolution checks go from hourly to every 3h — acceptable since HUMAN_ACTIVE mode means most runs take no action anyway
- **Mitigation**: If issues arise, cron can be reverted in a single commit
