# Proposed Change: Research Log Compaction for Routine Entries

## Problem
The research_log.md has grown to 889 lines and adds ~96 entries/day (4 sources checked hourly × 24 hours). Over 90% of entries are repetitive "no action" lines for unchanged sources, e.g.:
- `tokenman | routine state commit (...) | no action`
- `quarto-dev/quarto-cli | unchanged (...) | no action`
- `openai-harness-blog | Cloudflare-blocked (consistent) | no action`
- `pipeline-health | 10 failed runs checked, ALL ALREADY-FIXED | no action`

At current growth rate, the file will exceed 2000 lines within 12 days and become unreadable.

## Proposed Solution
Modify the evolve.yml research logging to use a compaction pattern:
1. For each source, only log when the state **changes** (new commit, new release, new failure)
2. When no change: increment a counter on the last entry instead of appending a new line
3. Format: `2026-03-30T18:00Z | source | unchanged (24h, 24 checks) | no action`
4. On state change: log the full new entry as today, resetting the counter

## Expected Impact
- Reduce research_log growth from ~96 entries/day to ~4-8 entries/day (only actual changes)
- ~90% reduction in log file growth
- Preserve all actionable information (changes are still fully logged)
- Same pattern as agent log compaction (PR #35, approved)

## Risk
- LOW: Read-only optimization, no behavioral change to research logic
- Research findings are still fully logged when they occur
- Only "no action" repetitions are compacted

## Implementation
- Modify evolve.yml research section to check previous entry before appending
- Use `tail -1` + grep pattern match to detect unchanged state
- Requires workflow YAML change → PR with needs-review label
