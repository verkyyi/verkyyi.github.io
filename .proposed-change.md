# Proposed Change: Watcher Health-Check Abbreviated Format

## Summary
Reduce watcher "all clear" health-check entries in agent_log.md from ~500+ character verbose format to ~100 character abbreviated format.

## Current Format (verbose)
```
2026-03-31T20:50Z | watcher.yml | health check | all clear — 0 corrective actions. Self-Evolve: 1 new transient failure at 20:14Z... pages-build-deployment (19:52Z), Weekly Analysis (18:21Z), Growth Strategist (18:22Z), Reviewer Agent (02:22Z) all healthy. Coder Agent: last success 11:51Z 03-27. Triage: last success 20:49Z 03-27. 0 open pipeline-fix issues. Token: 388pts, opus-4-6, 0 max-turns, 0 rate-limit. Evolve turns 22-26/30 (healthy). 6 PRs all needs-human. 2 issues (#24, #2) both triaged, needs-human. No broken chains, stuck runs, or recently closed issues. System stable.
```

## Proposed Format (abbreviated, for all-clear only)
```
2026-03-31T20:50Z | watcher.yml | all-clear | 7w healthy, 6p needs-human, 2i needs-human, 394t
```

Key: `Nw` = N workflows healthy, `Np` = N PRs status, `Ni` = N issues status, `Nt` = token data points.

## Rationale
- Watcher runs ~15x/day, producing 319 commits/week (45.8% of all commits)
- "All clear" entries are 95%+ of watcher logs and contain identical information
- Verbose format repeats workflow timestamps that change every hour — not useful in historical log
- Abbreviated format preserves key health signals (workflow count, PR/issue status, token health)
- Corrective-action entries would KEEP the verbose format (they contain actionable detail)
- Estimated reduction: ~80% fewer characters in watcher log entries, improving readability of agent_log.md

## Implementation
Modify the watcher.yml workflow prompt to instruct the agent to use abbreviated format when all checks pass and 0 corrective actions taken. Keep verbose format for any run with corrective actions, anomalies, or failures.

## Risk
Low. Only changes log formatting, not health-check behavior. Verbose detail still available for anomaly entries. Historical entries unaffected.

## Tier
PR — label: auto-merge (minor skill improvement, low-risk, no behavioral change to health checks)
