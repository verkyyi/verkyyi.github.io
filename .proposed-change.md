## Proposed Issue

**Title:** Implement evolve quiet mode — suppress HUMAN_ACTIVE no-op logging

**Labels:** priority:P1, needs-coder

**Body:**
The evolve workflow currently logs every HUMAN_ACTIVE no-op run (~24/day). This is the dominant contributor to agent_log growth (~18 lines/hr), causing rapid regrowth after truncation.

**Requested change:** Modify the evolve.yml workflow prompt to add hour-based filtering for HUMAN_ACTIVE no-op log entries:
- Log HUMAN_ACTIVE no-ops **only** at hours 00, 06, 12, and 18 UTC (4 entries/day instead of 24)
- Always log if there is a state change, action taken, or error (regardless of hour)
- Include a counter in the logged entry showing how many no-op runs were suppressed since last log

**Expected impact:** ~83% reduction in evolve log entries during idle periods (24→4 entries/day). Combined with watcher silent-clear (#53) and daily-digest (#64), total log growth would drop from ~18 lines/hr to ~4 lines/hr.

**Implementation path:** Coder agent should edit the evolve.yml prompt directly on main branch (AUTO-tier for state file updates, PR for workflow YAML changes).

## Risk
Low — only affects logging verbosity, not functionality. Evolve behavior unchanged. Quiet hours can be adjusted later.
