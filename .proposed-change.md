# Proposed Change: Fix cron frequency — evolve and watcher still running hourly

## Problem
PR #15 (merged 2026-03-26T07:56Z) was intended to reduce evolve.yml and watcher.yml cron frequency from hourly to every 3 hours. However, the cron expressions are still:
- evolve.yml: `'0 * * * *'` (every hour at :00)
- watcher.yml: `'30 * * * *'` (every hour at :30)

This results in ~48 state commits/day (96.6% of all commits), causing merge conflicts on every PR branch. This is the 5th consecutive week this issue has been identified and remains the root cause of the human-dependency bottleneck.

## Proposed Fix
Change cron expressions to every 3 hours:
- evolve.yml: `'0 */3 * * *'` (every 3 hours at :00)
- watcher.yml: `'30 */3 * * *'` (every 3 hours at :30)

Expected result: ~16 state commits/day (down from ~48), reducing merge conflict frequency by ~67%.

## Files Changed
- `.github/workflows/evolve.yml` line 5: `'0 * * * *'` → `'0 */3 * * *'`
- `.github/workflows/watcher.yml` line 5: `'30 * * * *'` → `'30 */3 * * *'`

## Risk
LOW — only changes scheduling frequency, no behavioral changes. Watcher health checks will still catch issues within 3h instead of 1h. Self-evolve still runs on every push to main (workflow_dispatch), so urgent changes are still handled immediately.
