# Proposed Change: FEATURE_STATUS.md as direct AUTO-tier commit

## Problem
FEATURE_STATUS.md has been recommended for 7 consecutive weeks. 7+ PR attempts
have ALL failed (#26, #27, #28, #29, #30, #31, #32 — rejected by reviewer or
misleading merges). The PR-based approach cannot succeed because:
1. State commits on main create merge conflicts before PRs can be reviewed
2. Reviewer correctly rejects PRs that contain only state changes alongside the
   new file
3. PR #29 "merged" but only deleted .proposed-change.md — did not create the file

## Solution
The evolve workflow should create `state/FEATURE_STATUS.md` via **direct commit
to main** (AUTO tier), not via PR. Per CLAUDE.md autonomy rules, FEATURE_STATUS
updates are explicitly listed as AUTO — commit directly, no PR needed.

## Changes Required
The evolve workflow's FEATURE_STATUS creation logic should:
1. Write `state/FEATURE_STATUS.md` directly to the working tree
2. Stage and commit it with message: `state: create FEATURE_STATUS.md`
3. Push to main — same as all other state file updates

No workflow YAML changes needed. This is a behavioral change in how the evolve
agent handles FEATURE_STATUS creation — it should use the same direct-commit
path it already uses for project_state.md and agent_log.md updates.
