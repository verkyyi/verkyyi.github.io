# Proposed Change: Research Log Quiet-Run Aggregation

## Problem
The research log (state/research_log.md) is at 1043 lines and growing ~96 lines/day. Each hourly watcher/evolve run appends 4 separate entries (one per source: tokenman, quarto-cli, openai-blog, pipeline-health), even when all sources return "no action". Over 90% of entries are routine "no action" / "unchanged" results.

## Proposal
When all research sources in a single run return "no action" (no new releases, no actionable findings, no new failures), log a single aggregated line instead of 4 separate entries:

**Before (4 lines per quiet run):**
```
2026-04-01T09:30:56Z | verkyyi/agentfolio | routine state commit, no new release | no action
2026-04-01T09:30:56Z | quarto-dev/quarto-cli | unchanged from prior runs | no action
2026-04-01T09:30:56Z | openai-harness-blog | Cloudflare-blocked (consistent) | no action
2026-04-01T09:30:56Z | pipeline-health | 10 failed runs checked, ALL ALREADY-FIXED, 0 actionable | no action
```

**After (1 line per quiet run):**
```
2026-04-01T09:30:56Z | research-check | all-quiet | tokenman:routine quarto:unchanged openai:blocked pipeline:0-actionable
```

When any source has a finding (new release, new commit of interest, actionable failure), that source gets its own full-detail entry as before. Only the "no action" sources are condensed.

## Impact
- Reduces research log growth by ~75% (~96 → ~24 lines/day for quiet periods)
- Preserves all actionable findings at full detail
- Source summaries in aggregated line remain searchable (grep-friendly)
- Follows established pattern: agent log compaction (PR #35) and abbreviated health checks (PR #38)

## Scope
- Affects: evolve.yml and watcher.yml research logging behavior
- Does not modify existing entries — forward-looking format change only
- Tier: minor skill improvement (auto-merge)
