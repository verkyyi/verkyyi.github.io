# PDF Export Design

## Summary

Add PDF export to AgentFolio using the JSON Resume ecosystem. Each adapted JSON Resume file (`data/adapted/{slug}.json`) gets a corresponding PDF (`data/adapted/{slug}.pdf`) generated by `resumed` with `jsonresume-theme-onepage`. A dedicated GitHub Actions workflow handles generation, and the site renders a download button when a PDF is available.

## Architecture

### Pipeline

```
adapt.yml:  data/input/** changes â†’ generates JSON â†’ commits to data/adapted/
pdf.yml:    data/adapted/*.json changes â†’ generates PDFs â†’ commits to data/adapted/
```

PDF generation is fully decoupled from adaptation. The PDF workflow triggers on any change to `data/adapted/*.json`, regardless of source.

### PDF Generation

- **Tool:** `resumed` (npm package) â€” lightweight JSON Resume renderer
- **Theme:** `jsonresume-theme-onepage` â€” compact single-page layout optimized for PDF
- **Input:** `data/adapted/{slug}.json`
- **Output:** `data/adapted/{slug}.pdf`
- **Selective generation:** Only regenerate PDFs for changed JSON files (same pattern as adapt.yml)

### Workflow: `.github/workflows/pdf.yml`

**Triggers:**
- `push` to `main` with changes to `data/adapted/*.json`
- `workflow_dispatch` with optional `slugs` input (space-separated, empty = all)

**Steps:**
1. Checkout
2. Detect changed JSON files (extract slugs from changed paths; workflow_dispatch uses input or all)
3. Install `resumed`, `jsonresume-theme-onepage`, and `puppeteer`
4. For each slug, run `resumed export data/adapted/{slug}.json -t jsonresume-theme-onepage -o data/adapted/{slug}.pdf --puppeteer-arg=--no-sandbox`
5. List generated PDFs
6. Commit and push (`chore(pdf): regenerate PDFs`)

**Committer:** AgentFolio Bot (same as adapt.yml)

### Download Button

- `DownloadPdf` component â€” fixed-position link in the top-right corner
- Does a HEAD request to `data/adapted/{slug}.pdf` â€” renders only if PDF exists
- Hidden during print (`@media print { display: none; }`)
- No error state, no loading spinner needed

### Local Development

`npm run copy-data` already syncs `data/adapted/` â†’ `web/public/data/adapted/`, so locally generated PDFs are available at `/data/adapted/{slug}.pdf` without extra config.

## Decisions

- **PDFs are committed** to the repo alongside JSON, same as adapted JSON files. They are generated artifacts.
- **Theme choice:** `jsonresume-theme-onepage` â€” compact, single-page, professional. Does not render the projects section (acceptable trade-off for single-page fit).
- **Theme compatibility:** Work entries include both `name` and `company` fields â€” `name` is the JSON Resume standard, `company` is what the onepage theme expects.
- **Puppeteer sandbox:** `--no-sandbox` flag required on GitHub Actions runners.
- **No server-side rendering.** No changes to the Vite build.

## Out of Scope

- Custom JSON Resume theme matching the site pixel-for-pixel
- Print CSS improvements to ResumeTheme.tsx
- Serverless / on-demand PDF generation API
- Projects section in PDF (onepage theme limitation, acceptable for senior-level resumes)
