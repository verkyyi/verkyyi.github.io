# Pipeline Separation â€” Fit, Extract, Structurize

Split the monolithic `/adapt` skill into a three-stage pipeline with human-editable intermediate output and automatic directive extraction.

## Motivation

The current `/adapt` skill does two things in one shot: converts freeform `resume.md` into a valid JSON Resume structure AND tailors content for a target role. This coupling means:

- No opportunity for human review or editing before the final output
- Schema conformance and creative tailoring are entangled in one prompt
- Customization preferences are lost between runs

Separating the pipeline lets humans edit tailored content in familiar markdown, builds up a durable directives file from their edits, and keeps schema conversion as a clean final step.

## Pipeline Overview

```
data/input/resume.md           <- source of truth (human-authored)
data/input/directives.md       <- global preferences (human + auto-extracted)
data/input/jd/notion.md        <- target JD

         /fit
          |
          v
data/fitted/notion.md          <- tailored freeform markdown (human-editable)

      human edits (optional)
          |
          v
     /extract-directives       <- diffs fitted vs last bot commit
          |
          v
data/input/directives.md       <- updated with extracted intentions

     /structurize
          |
          v
data/adapted/notion.json       <- final JSON Resume for rendering + PDF
data/adapted/notion.pdf        <- generated by existing pdf.yml workflow
```

Directive extraction only runs when there are actual human edits to fitted files. If the user didn't touch the fitted markdown, there's nothing to extract.

## Skills

### `/fit`

Tailoring skill. Reads the source resume, a target JD, and the global directives file. Writes tailored freeform markdown.

- **Reads:** `data/input/resume.md` + `data/input/jd/{slug}.md` + `data/input/directives.md`
- **Writes:** `data/fitted/{slug}.md`
- **Output format:** Markdown, same structure as `resume.md` but rewritten and reordered for the target role. Highlights rephrased, sections reordered, summary tailored. Directives applied.
- **No JD (default):** Reads resume + directives only, writes `data/fitted/default.md` as a general-purpose version.
- **Invocation:** `/fit` (all JDs + default), `/fit notion` (one slug)

### `/extract-directives`

Intention extraction skill. Diffs human-edited fitted files against the last bot-generated version and appends extracted intentions to the global directives file.

- **Reads:** `data/fitted/{slug}.md` (current state) + last bot-authored commit of same file (via git)
- **Writes:** Appends to `data/input/directives.md`
- **Mechanism:** Uses git authorship to distinguish bot output from human edits. Finds the last commit to each fitted file authored by `AgentFolio Bot`, diffs the current file against that version. Any changes since the bot's commit are human edits.
- **Output format:** Extracted intentions, not literal diffs. E.g., "Prefer 'platform engineer' over 'backend engineer' in summary", "Keep volunteer section brief."
- **Invocation:** `/extract-directives` (all fitted files with human changes), `/extract-directives notion` (one slug)
- **No-op:** If no fitted files have human edits (no diff against last bot commit), exits silently.

### `/structurize`

Schema conversion skill. Reads tailored freeform markdown and writes a valid JSON Resume document.

- **Reads:** `data/fitted/{slug}.md`
- **Writes:** `data/adapted/{slug}.json`
- **Output:** Valid JSON Resume. Minimal meta: `company`, `role`, `generated_by`, `lastModified`. No match scores, skill emphasis, or section order.
- **Invocation:** `/structurize` (all fitted files), `/structurize notion` (one slug)

## CI Workflows

Four workflows, chained via push triggers:

### `fit.yml`

- **Trigger:** Push to `data/input/resume.md`, `data/input/jd/**`, or `.claude/skills/fit.md`. Also `workflow_dispatch`.
- **Does NOT trigger on** `data/input/directives.md` changes (avoids loop: extract updates directives, which would re-trigger fit, which would overwrite fitted files).
- **Steps:** Install Claude Code, run `/fit`, commit as `AgentFolio Bot <bot@agentfolio.local>`, push.

### `extract.yml`

- **Trigger:** Push to `data/fitted/*.md` by a non-bot author.
- **Author check:** Skips if the commit author is `AgentFolio Bot` (those are pipeline-generated, not human edits).
- **Steps:** Install Claude Code, run `/extract-directives`, commit updated `directives.md` as `AgentFolio Bot`, push, then trigger `structurize.yml` via `workflow_dispatch`.

### `structurize.yml`

- **Trigger:** Push to `data/fitted/*.md` by bot author, OR `workflow_dispatch` (triggered by `extract.yml` after extraction completes).
- **Author check (push trigger only):** Only runs if the commit author is `AgentFolio Bot` (from `/fit`).
- **Steps:** Install Claude Code, run `/structurize`, commit as `AgentFolio Bot`, push.

### `pdf.yml` (unchanged)

- **Trigger:** Push to `data/adapted/*.json`.
- **Steps:** `resumed export` to generate PDFs, commit as `AgentFolio Bot`, push.

### Chain behavior

- **Resume/JD change:** push triggers `fit.yml` (bot commit to fitted/) -> triggers `structurize.yml` (bot commit to adapted/) -> triggers `pdf.yml`.
- **Human edits fitted markdown:** push triggers `extract.yml` (bot commit to directives.md, then triggers structurize.yml via workflow_dispatch) -> `structurize.yml` runs (bot commit to adapted/) -> triggers `pdf.yml`.
- **Manual directives edit:** No auto-trigger. User runs `workflow_dispatch` on `fit.yml` to re-fit with updated directives.

### Author detection in workflows

Workflows check commit author to determine bot vs human:

```yaml
- name: Check commit author
  id: author
  run: |
    AUTHOR=$(git log -1 --format='%an')
    echo "author=$AUTHOR" >> "$GITHUB_OUTPUT"
    echo "is_bot=$( [ "$AUTHOR" = "AgentFolio Bot" ] && echo true || echo false )" >> "$GITHUB_OUTPUT"
```

`extract.yml` runs when `is_bot=false`. `structurize.yml` runs when `is_bot=true`.

## Local Workflow

### Full pipeline, no edits:

```
/fit notion

```

### With human editing:

```
/fit notion
# edit data/fitted/notion.md
/extract-directives
/structurize notion
```

### Directives-only (no JD changes):

```
# edit data/input/directives.md
/fit
/structurize
```

## Directives File Format

`data/input/directives.md` is a freeform markdown file. Humans can write directives directly. The `/extract-directives` skill appends auto-extracted sections:

```markdown
# Resume Directives

## Manual
- Always lead with data infrastructure experience
- Prefer "platform engineer" over "backend engineer"

## Auto-extracted (2026-04-16, from notion.md edits)
- Shorten volunteer section to one line
- Emphasize Kafka and Flink experience in highlights
```

## Out of Scope

- `meta.agentfolio` fields beyond `company`, `role`, `generated_by`, `lastModified` (match scores, skill emphasis, section order are unused by the frontend â€” add back when a UI feature needs them)
- Changes to `ResumeTheme.tsx` or any frontend rendering code
- Changes to `pdf.yml` workflow
- Pre-commit hooks (extraction is manual locally, CI is the safety net)
