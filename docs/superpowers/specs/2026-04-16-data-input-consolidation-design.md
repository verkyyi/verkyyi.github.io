# Data Input Consolidation

Consolidate all user-editable files into `data/input/` so personalizing AgentFolio is "replace one folder."

## Motivation

Currently `data/` mixes user-input files (`resume.md`, `jd/`) with auto-generated outputs (`adapted/`, `slugs.json`, `llm_cache/`). New users must figure out which files to replace and which to leave alone. Moving all user input into a single subfolder makes personalization a one-step operation.

## New Folder Structure

```
data/
├── input/                  # USER EDITS THIS
│   ├── resume.md           # Resume in any text format (source of truth)
│   └── jd/                 # One .md per target role (filename = URL slug)
│       └── sample-company.md
├── adapted/                # Auto-generated by adapt workflow
│   ├── default.json
│   └── sample-company.json
├── slugs.json              # Auto-generated by adapt workflow
└── llm_cache/              # Auto-generated LLM response cache
```

## File Moves

| From | To |
|---|---|
| `data/resume.md` | `data/input/resume.md` |
| `data/jd/` | `data/input/jd/` |

## Changes by Area

### Python Scripts

**`scripts/adapt_from_markdown.py`** — Update `data_dir` references:
- Resume path: `data/resume.md` -> `data/input/resume.md`
- JD glob: `data/jd/*.md` -> `data/input/jd/*.md`

**`scripts/chat_answer.py`** — Update default `--resume` argument:
- `data/resume.md` -> `data/input/resume.md`

**`scripts/fetch_jds.py`** — Update company profiles path (line 116):
- `data/companies/` -> `data/input/companies/`

### GitHub Actions Workflows

**`.github/workflows/adapt.yml`** — Update trigger paths:
- `data/resume.md` -> `data/input/resume.md`
- `data/jd/**` -> `data/input/jd/**`

**`.github/workflows/jd-sync.yml`** — Update `data/companies` references (lines 27, 31) to `data/input/companies`.

### Web Build Pipeline

**`web/package.json` `copy-data` script** — Instead of copying all of `data/`, copy only what the web app needs:

Before:
```js
node -e "require('node:fs').cpSync('../data', 'public/data', {recursive: true})"
```

After:
```js
node -e "const fs = require('node:fs'); fs.cpSync('../data/adapted', 'public/data/adapted', {recursive: true}); fs.cpSync('../data/slugs.json', 'public/data/slugs.json');"
```

This prevents raw resume markdown, JDs, and LLM cache from being served publicly.

### No Changes Needed

- **Web app runtime** (`useAdaptation.ts`, `useVisitorContext.ts`) — fetches `data/adapted/{slug}.json` and `data/slugs.json` at the same public URL paths, unaffected.
- **`deploy.yml`** — calls `npm run build` which invokes the updated `copy-data`.
- **Tests** — mock the same fetch URLs, unaffected.

### Documentation

- **`CLAUDE.md`** — Update project structure tree and personalization instructions.
- **`README.md`** — Update project structure tree and "How to Personalize" section.

## Scope

~8 files changed. Zero runtime behavior change for the web app. Auto-generated outputs remain committed in `data/` alongside `input/`.
