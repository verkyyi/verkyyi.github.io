# AgentFolio

Open-source agentic portfolio engine. See `README.md` for setup and personalization.

## Derived repo rule (this repo)

This is a **derived repo** — the framework was copied in from the upstream
project `verkyyi/agentfolio`. In this repo, **only the `data/` folder may
be modified**. Do not edit `web/`, `.github/`, `.claude/`, scripts, configs,
or any framework source. Framework fixes belong upstream in `verkyyi/agentfolio`;
re-apply them here by re-running the deploy from the latest upstream.

The only exception: `CLAUDE.md` and `README.md` may carry deploy-specific notes
(like this one), but their framework sections should track upstream.

## Routing

Slugs live at the root — no `/c/` prefix:

```
/                → data/adapted/default.json
/notion          → data/adapted/notion.json
/dashboard       → Owner-only dashboard (not linked from public pages)
/unknown         → 404 page
```

The slug is the first path segment after the base path. `/dashboard` is intercepted in `App.tsx` before slug resolution. All other slugs are resolved by `useAdaptation` which fetches `data/adapted/{slug}.json`. Unknown slugs return a 404 — no silent fallback.

## Dashboard

Owner-only readonly dashboard at `/dashboard` for reviewing fitted resumes. Not linked from public pages — accessed by typing the URL directly.

**Layout:** Sidebar + content area. Sidebar lists fitted resumes (with labels from fit-summary) and a Directives viewer under Settings.

**Tabs per resume:** Preview (rendered markdown with fit-summary card), Diff (inline word-level diff vs default), PDF (embedded viewer), JD (target job description).

**Tab bar actions:** URL copy input, Open link, PDF download.

**Fit summary:** Each fitted markdown can have an HTML comment at the top (`<!-- fit-summary: ... -->`) with `target` and `changes` fields. The `/fit` skill generates these. `parseFitSummary` extracts them; `copy-data` uses them for sidebar labels.

## Key Conventions

- **Test framework:** Vitest, not Jest. Use `vi.fn()`, `vi.mock()`, etc.
- **Env vars:** `import.meta.env.VITE_*` in browser code, `loadEnv()` in vite.config.ts.
- **Resume source of truth:** `data/input/resume.md` — any text format.
- **Pipeline:** `/fit` (resume + JD + directives → `data/fitted/*.md`) → `/extract-directives` (human edits → directives) → `/structurize` (fitted markdown → `data/adapted/*.json`).
- **Fitted resumes:** `data/fitted/` — tailored markdown, human-editable. Generated by `/fit`, consumed by `/structurize`.
- **Directives:** `data/input/directives.md` — global preferences. Written by humans and auto-extracted by `/extract-directives`.
- **Adapted output:** `data/adapted/` — auto-generated JSON Resume + PDFs. Do not edit manually.
- **Resume theme:** `ResumeTheme.tsx` renders any valid JSON Resume document using styled-components. Based on jsonresume-theme-developer-mono. Supports all 12 JSON Resume sections.
- **PDF theme:** `jsonresume-theme-onepage` via `resumed` CLI. Compact single-page layout. Does not render projects section.
- **PDF compat:** Work entries include both `name` and `company` fields — `name` is JSON Resume standard, `company` is what the onepage theme expects.
- **Fit summary format:** `<!-- fit-summary: ... -->` HTML comment at top of fitted markdown with `target` (company + role) and `changes` (3-6 bullet points). Generated by `/fit`, parsed by dashboard.
- **Build pipeline:** `npm run copy-data` (via `web/scripts/copy-data.cjs`) syncs `data/adapted/`, `data/fitted/`, `data/input/jd/`, and `data/input/directives.md` to `web/public/` and generates `data/fitted/index.json` with labels. Runs before every build/dev start.
- **Deploy trigger:** Bot workflows (fit, structurize, pdf) explicitly trigger deploy via `gh workflow run deploy.yml` after committing, since `GITHUB_TOKEN` pushes don't trigger `on: push` workflows.
